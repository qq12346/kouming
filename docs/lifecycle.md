## 1. 产品基因

### 一句话

叩鸣·工坊是一个 AI Agent 编排工坊——你定义意图，多个 AI Agent 协作规划、执行、审查，产出知识工作成果。同时追问你"这个意图值得吗？"

### 竞争坐标

| 竞品 | 类 | 叩鸣与之不同 |
|------|-----|------------|
| CrewAI / AutoGen | 多Agent·效率至上 | 叩鸣有价值观立场（宪法） |
| ChatGPT / Claude | 单轮对话·无编排 | 叩鸣多Agent装配线 |
| Cursor / Codex | 单Agent·代码沙箱 | 叩鸣产品类别不同，但借鉴 execpolicy 架构 |
| Devin | AI 替你做完 | 叩鸣优化"你做你自己的同时 AI 替你做更多" |

### 三条宪法（不可拔插的系统底座）

| 宪法 | 来源 | 含义 |
|------|------|------|
| 尊严宪法 | 康德 | AI 必须声明身份，用户有权知道 AI 参与 |
| 自主宪法 | 存在主义 | 每个 AI 输出必须附带至少 2 个替代方案 |
| 追问宪法 | 苏格拉底 | AI 输出必须标注前提假设，供用户审视 |

### 四条 Agent

| Agent | 职责 |
|-------|------|
| Planner | 拆解用户意图为 2-5 个子任务 |
| Researcher | 为每个子任务搜索和整合信息 |
| Creator | 生成内容（每个子任务独立调用） |
| Reviewer | 审查整体输出质量（1-5 评分 + 问题列表） |

### 代偿审计（王东岳）

"你的 AI 越来越好了，你越来越不好了？"

- 能力保留度：自己做 + 反驳 AI + 反思 / 总任务
- AI 介入度：AI 参与决策的程度（越低越好）
- 追问频率：溯源完成率 + 暂停反思频率

## 3. 技术架构

### 当前技术栈（v0.1）

```
前端：React + Vite + Tailwind
状态：Zustand（persist → localStorage）
AI：Vercel AI SDK（DeepSeek API / OpenAI 兼容）
路由：React Router v7
测试：Vitest
部署：CloudBase 静态托管
存储：localStorage（API Key / 任务历史 / 审计数据）
```

### v0.3 变更

```
+ Electron 壳
+ API Key → macOS Keychain / Windows Credential Manager
+ 本地 SQLite（替代 localStorage 持久化）
+ 本地向量数据库（LanceDB / 纯内存方案）
+ npm / brew 分发
```

### 技术决策记录

| 决策 | 时间 | 理由 |
|------|------|------|
| BYOK 模式 | 2026-07-10 | AI 信息流不经过我方服务器，合规风险最低 |
| 不舒服模式默认关闭 | 2026-07-12 | 老板决定 |
| 定位修正为"Agent编排工坊" | 2026-07-12 | 不再对标 Codex，不再对标聊天机器人 |
| Electron v0.3 | 2026-07-12 | 本地软件 = 软件销售，ICP 仅需备案 |
| 纵向紧耦合+横向松耦合 | 初始 | 宪法→所有Agent 紧耦合，Agent间消息 松耦合 |
