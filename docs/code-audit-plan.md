## 问题 1：Agent 硬编码模型

**现状**：4 个 Agent 的 `model: deepseek('deepseek-v4-pro')` 是硬编码的。
**影响**：如果 DeepSeek 涨价/停服/用户想用别的模型，所有 Agent 全部瘫痪。
**修复**：

```
Day 1：
1. userStore 新增字段：modelProvider（'deepseek' | 'custom'）、customBaseURL、modelName
2. agentStore/agents/ 从 userStore 读取模型配置而非硬编码
3. Settings.jsx 新增模型选择 UI（DeepSeek / 自定义 OpenAI 兼容端点）
```

**验收**：设置页可切换模型，Agent 使用选定的模型运行。

---

## 问题 2：守护进程无 UI 接收器 🔴

**现状**：`guardian/index.js` 通过 `GuardianEvents`（EventTarget）发出通知，但没有任何 React 组件监听。
**影响**：五项监控（能力退化/想像力衰减/追问缺失/指标漂移/尊严红线）在后台运行但用户完全看不到——守护进程是"盲人"。
**修复**：

```
Day 2：
1. guardanStore 新增字段：alerts[]（由守护进程写入）
2. guardian/index.js 的 EventTarget 改为直接写入 guardianStore.setAlert()
3. AssemblyLine.jsx 右侧语境面板新增"守护进程消息"区域
4. 显示逻辑：
   - 能力保留度 < 30 → 🟡 "你今天更多在让AI替你思考"
   - 追问频率 < 20% → 🟡 "你今天几乎没有追问——所有假设都接受了"
   - 指标漂移（7天对比） → 📊 "过去7天你的能力保留度下降了X%"
```

**验收**：执行任务后，AssemblyLine 右侧面板能看到守护进程的实时反馈。

## 问题 3：审计无历史对比

**现状**：`audit/collector.js` 记录的是内存变量，刷新后从零开始。
**影响**：审计面板只能看"这次打开以来"的指标——无法对比"上次任务"或"上个月"的状态。
**修复**：

```
Day 3-4：
1. auditStore（新 Zustand store）+ persist 中间件
2. AuditCollector 每次 `getReport()` 时将结果写入 auditStore.history[]
3. AuditPanel.jsx 新增"历史"Tab：
   - 按日期查看每次任务的 AI 介入度/能力保留度/追问频率
   - 趋势图（7天/30天）
   - 漂移警告（7天内能力保留度下降超过30%）
```

**验收**：多次运行任务后，审计页的历史 Tab 显示日期列表和趋势图。

## 问题 4：宪法正则逃逸

**现状**：三条宪法全用正则匹配。AI 可以在不语义满足条件的情况下通过检测。
**影响**：AI 输出"AI 参与"四个字但不由 AI 生成 → 宪法判"通过"。AI 输出"替代方案替代方案"但不提供任何实质替代 → 也通过。
**修复**（优先级低——当前够用）：

```
Day 5：
方案 A（保持正则但加固）：
- 尊严：不只是检查"AI参与"，还检查是否有 叩鸣·工坊 的署名格式
- 自主：替代方案关键词 + 至少2行结构化列表（以数字或 - 开头）

方案 B（语义级，依赖 AI SDK 能力）：
- 调用第二遍 AI 做判断："这段文字真的声明了AI身份吗？"
- 成本：每条输出再多调一次 API
- 暂不实施——待用户反馈
```

**验收**：尊严宪法：伪造的"AI 参与"文本被识别。自宪法：格式化的替代方案列表被识别。

## 问题 5："自主"标准未传达

**现状**：用户打开 Dashboard → 输入意图 → 配价值观 → 运行——但整个过程没有向用户解释"为什么这三条宪法存在"和"什么是好的自主"。
**影响**：用户可能会把叩鸣当普通AI工具用——不理解"自己做"按钮的意义、"不舒服模式"的意义、审计面板的意义。
**修复**：

```
Day 6：
1. Dashboard 确认卡底部新增一行解释文案（<100字）
2. 每个价值观选择旁加 tooltip："选择'快速'意味着你优先效率，但叩鸣会帮你盯着。"
3. AssemblyLine 第一次运行时弹引导提示（仅首次）：
   "这是你的装配线。每个Agent的输出都经过三条宪法检查。
   看到'自己做'按钮了吗？那是你随时可以拿回控制权的地方。"
```

**验收**：新用户首次运行时看到引导提示。Dashboard 有解释文案。

## 问题 6：简单任务过度加工

**现状**：编排引擎硬编码 Planner → 所有任务都被拆解为子任务。
**影响**：简单任务被 Planner 过度拆解，浪费 API 调用和用户时间。
**修复**：

```
Day 7：
1. 用户可在 Dashboard 选择"快速模式"——跳过 Planner，直接用 Creator
2. 编排引擎检查：如果 intent.complexity === 'simple'，跳过 Planner + Researcher
3. 简单模式下：直接走 Creator → Reviewer
```

**验收**：简单模式下一个简短意图直接生成结果，不经过子任务拆解。

---

## 问题 7：任务历史

**现状**：用户执行完任务后，切到别的页面再回来——只能看到最后一次任务的结果。没有"之前做了哪些任务"的列表。
**影响**：用户无法回顾、对比、或重新审视过去的输出。
**修复**：

```
Day 8（用现有的 agentStore.result 数据）：
1. agentStore 新增 taskHistory[]（每个已完成的任务 push 进去）
2. Dashboard 新增"历史任务"区域：日期 + 意图摘要 + 状态（已完成/已中断）
3. 点击历史任务 → 跳转到 AssemblyLine 并加载该任务的 result（已持久化）
```

**验收**：Dashboard 显示历史任务列表。点击可查看过去的执行结果。
